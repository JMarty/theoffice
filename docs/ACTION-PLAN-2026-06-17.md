# The Office — kód-audit és akcióterv (2026-06-17)

**Vezetői összefoglaló.** A kód érdemben egészséges: az SQL mindenhol paraméterezett (nincs injection), a token-összevetés `timingSafeEqual`, a DB 0600-zal jön létre, a `/tenant/**` gitignore vízhatlan, a single-writer queue + egyetlen deliverer a kódbázis legjobb architektúra-döntése (gemini hozzáadása tényleg egy modul volt). A fő kockázat nem biztonsági, hanem **megbízhatósági**: az aszinkron runtime-ok (codex/gemini) a gyenge mag — nincs codex turn-timeout/kill watchdog, nincs boot-reaper a `delivering` sorokra, a gemini exit-0-ra sikert jelez, és a claude deliverer tick nem reentrancy-védett. Ezek "megkapta de nem reagált" típusú néma hibák. A korábbi audithoz képest a rate-limit lockout-bug és a login-flash javítva van; a getClientIp spoof és az XSS-osztály viszont nyitva (utóbbi most LOW, mert nincs élő kihasználható út). Egyetlen ideghalál sincs CRITICAL szinten — ez egytulajdonos, owner-trusted, saját gépen futó telepítés.

8 független terület-audit (web-server-auth, web-ui-frontend, runtime-layer, data-queue-config, features, ops-scripts-build, testing-coverage, architecture-crosscutting) készült a jelenlegi kódról, majd **minden HIGH/MEDIUM+ finding adverzáriálisan újra-verifikálva** lett a lemezen lévő kód ellen (a verifier több empirikus reprodukciót is futtatott: Node v22 `req.destroy()` viselkedés, FTS5 sentinel, DST tickek, memória-recall saturation). Ahol a finder súlyozása túlzott volt, a `severityFinal`-t használtam és megindokoltam. Kereszthivatkozás a korábbi `docs/CODE-AUDIT-zeus-2026-06-15.md` audithoz az 5. szekcióban.

Nincs CI/lint/format/pre-commit.

## Téma szerinti bontás

**Biztonság.** Egytulajdonos, owner-trusted, minden agent `full` profil — ezért nincs CRITICAL. Élő gap: a getClientIp spoof (P1 #6, a futó process tényleg 0.0.0.0-n köt). Az `esc()` nem escape-eli `'`/backtick (`web-ui/app.js:7`) — **LOW**, mert minden single-quoted slot owner-kontrollált dir-nevet (`a.id`, src/agents.ts:40) vagy szerver-generált hexet (`c.id`, randomBytes) kap, és az LLM-írt free-text mind text-node kontextusban marad; a "raw style interpoláció" mint élő rés **refuted** (minden `style=` derived color/szám helperből jön). Javasolt mégis a kvótabiztos `esc()` (egysoros, az egész osztályt zárja) + opcionális CSP `script-src 'self'`. Write-endpoint validáció hiánya (`server.ts:306-394`, agentId/from/to nincs `loadAgents` ellen ellenőrizve) — **LOW** adatintegritás-footgun (orphan sor / sosem kézbesülő bus-üzenet), legalább a routing-endpointokat (from/to/agent) validáld. Slack file host-allowlist (`files.ts:35-38`, xoxb token verbatim URL-re) — **LOW** defense-in-depth, `new URL(url).hostname.endsWith('.slack.com')`. DM-only nincs kódban kényszerítve (`slack-ingest.ts:58-70`) — **LOW** misconfiguráció-footgun ha `channels:history` bekerül. `tenant/store` dir **0775** (a benne lévő secret-fájlok 0600-asok, de a dir traversálható) — **LOW**, `chmod 700` az install.sh-ban. A `--dangerous-*` flag profil-nem-kényszerítés codex/gemini-n: **LOW/informational** a korábbi audit saját korrekciója szerint — a flageket **ne** vedd ki (headless-hez kell), a Claude deny-lista enforce-olt; opcionális guardrail: warn/refuse, ha non-`full` profil codex/gemini agenten. Bootstrap `curl|bash` + `curl|sudo bash` pinning nélkül — **LOW**, owner-trusted norma; pin-elj taget (`git clone --branch vX.Y.Z`) és kínálj "review then run" alternatívát.

**Robusztusság/megbízhatóság.** Ez a fő gyengeség, lásd P0 #1-4 + P1 #5/#10 + P2 #16. Másodlagos: `inFlight` szivárgás szinkron throw-ra `inFlight.add` és a child-wiring között (`codex-runtime.ts:82-107`) — **MEDIUM**, a reális trigger szűk (`.env` ami directory vagy mode-000; a `.reply-context` writeFileSync már `try/catch`-elt, a spawn/ENOENT async error), javítás: `inFlight.add`/`markDelivering` sikeres spawn UTÁN. Dupla `fail()` `error`+`close` együtt-tüzelésre (`codex-runtime.ts:129-162`) — **LOW**, `let settled=false` latch. `.env` feltétel nélkül felülírja `PATH`/`HOME`/`TZ`/`OFFICE_*`-ot mindhárom runtime-on — **LOW** self-inflicted, alkalmazd a `.env`-et előbb, a reserved kulcsokat utána. claude `submit-give-up` requeue attempt-et éget (`claude-runtime.ts:160,189`) — **LOW**, használj `requeueNoPenalty`-t a tranziens útra. Shutdown fire-and-forget (`index.ts:48-56`, nem várja a ticket, nem öli a gyerek-processeket) — **LOW**, súlyosbítja a #2-t; bounded drain + runtime `shutdown()`.

Slack roster boot-snapshot (`slack-ingest.ts:108`, `slack-send.ts:21`) — **LOW**: a sanctioned `new-agent.sh:46-48` már restartol, csak a ritka **re-tokening** igényel kézi restartot. `product/` réteg dokumentált+bekötött de lemezen nem létezik (`config.ts:98`) — **LOW** koncepció-drift, vagy commitold a `product/config.example.json`-t, vagy egyszerűsítsd a README-t két rétegre.
